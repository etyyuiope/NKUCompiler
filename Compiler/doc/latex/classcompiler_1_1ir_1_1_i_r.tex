\doxysubsection{Detailed Description}
The class for \mbox{\hyperlink{classcompiler_1_1ir_1_1_i_r}{IR}} (Intermediate Representation). Some representation notations are taken from LLVM. 

\begin{DoxySeeAlso}{See also}
\href{https://llvm.org/docs/LangRef.html}{\texttt{ https\+://llvm.\+org/docs/\+Lang\+Ref.\+html}}
\end{DoxySeeAlso}
Examples\+: ADD i32 foo, i8 bar, double baz ALLOC i32, align 8.

\begin{DoxyNote}{Note}
LOGIC EXPLAINED When an compiler\+::\+IR instance is created from the AST, we should call compiler\+::ir\+::\+IR\+::walk\+\_\+ir() function to generate the string as follows\+:
\begin{DoxyEnumerate}
\item Create a lambda function that can be passed to compiler\+::ir\+::\+IR\+::walk\+\_\+ir() that does some job with current operation;
\item The function compiler\+::ir\+::\+IR\+::walk\+\_\+ir() will iterate through each operands, and because each operand can be another instance of \mbox{\hyperlink{classcompiler_1_1ir_1_1_i_r}{compiler\+::ir\+::\+IR}}, we need to dig into the ~\newline
 operand, where callback function comes to help.
\item The callback will call each operand with callback again until it reaches a deepmost level\+: the raw operand that we can ensure whether it is chained to the caller or not. 
\end{DoxyEnumerate}
\end{DoxyNote}
